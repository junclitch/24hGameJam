# 壁キックアクション 実装まとめ

実装日: 2026-05-05 (19:00 〜 20:30 頃)
対象シーン: `Assets/Scenes/ScrollAction.unity`

## 概要

ScrollAction の PlayerAction インベントリ系に「壁キック」アクションを追加した。`wall-jump.png` (11 フレーム) を使い、以下を実現:

- **空中で壁の近く (左右) にいる時にスペースを押す**と発動
- ウィンドアップ (壁張り付き) → キック実行 → アニメ完了の3段階 FSM
- キック実行時に **上昇速度 + 壁と反対方向の水平速度**（共に jumpForce 相当の 14 unit/sec）を適用
- アニメは 02 (壁に接触) → 04 (壁をキック) → 11 (着地) の 10 フレームを 12 fps で再生
- 壁の左右に応じて `flipX` を切替 (原画は「右壁を蹴って左に飛ぶ」向きなので、左壁時のみ反転)

実装は基本的に MCP for Unity 経由で完結。スプライトの再スライス・アニメクリップ作成・Animator state machine 編集まで全部スクリプト駆動で行った。

## 成果物

### 1. PlayerAction (新規)
`Assets/ScrollAction/Scripts/Actions/WallKickAction.cs`

- `[CreateAssetMenu]` 付き ScriptableObject (Crouch / Glider / Sliding と同じ A+B+C パターン)
- 内部状態: `enum Phase { Idle, Windup, Kicked }` + `elapsed` + `wallSide`
- `OnFixedTick` の流れ:
  1. **Idle**: `ctx.jumpRequested && !ctx.isGrounded && DetectWall() != 0` で `StartWallKick()` 呼出
  2. **Windup**: `rb.linearVelocity = (wallSide × 0.1, 0)` で壁張り付き、`elapsed >= windupDuration` で Kicked へ
  3. **Kicked**: `rb.linearVelocity = (-wallSide × kickH, kickV)` を一発適用、`elapsed >= windup + postKick` か `isGrounded` で Idle へ
- 壁検出 (`DetectWall`) は `Physics2D.Raycast` を左右に飛ばし、`PlayerStats.groundLayer` 流用
- `StartWallKick` で `ctx.jumpRequested = false` を立てて、同 Tick の JumpAction 二重発火を抑止

```csharp
private void StartWallKick(PlayerActionContext ctx, float side)
{
    wallSide = side;
    elapsed = 0f;
    phase = Phase.Windup;

    ctx.rb.linearVelocity = new Vector2(wallSide * 0.1f, 0f);

    ctx.jumpRequested = false;
    ctx.isWallKicking = true;
    ctx.wallKickSide = wallSide;
}
```

## アーキテクチャ的な選択

### 3段階 FSM (Idle / Windup / Kicked) にした理由

ユーザー要件は「開始は 02、04 で蹴り、05 以降は飛行」。これは「視覚的に 0.15 秒の溜めの後に蹴る」という時間構造を持つ。

- **単発キック方式** (Space で即 vx/vy を適用) だと視覚と物理がズレる: アニメは 02 から始まるが物理はもう蹴っている
- **3段階 FSM** にすると、Windup 中に物理を「壁張り付き」に固定 → アニメ的に蹴る瞬間 (frame_04) と物理キック適用が一致

windupDuration (0.15) はアニメの 02→03→04 の表示時間 (2/12 ≒ 0.167) とほぼ同期している。

### 壁検出に Physics2D.Raycast を使った理由

GroundCheckAction が `Physics2D.OverlapCircle` を使っているのと統一感は薄いが、壁検出は方向性が必要 (左/右どちらの壁か) なので Raycast が自然:

```csharp
var hitR = Physics2D.Raycast(origin, Vector2.right, wallCheckDistance, mask);
if (hitR.collider != null) return 1f;

var hitL = Physics2D.Raycast(origin, Vector2.left, wallCheckDistance, mask);
if (hitL.collider != null) return -1f;
```

`PlayerStats.groundLayer` を流用しているので、壁が地形と同じレイヤである前提。別レイヤ運用が必要なら `WallKickAction` に `[SerializeField] LayerMask wallLayer` を切り出して差し替える。

### jumpRequested の消費による JumpAction 抑止

WallKickAction が成立した瞬間に `ctx.jumpRequested = false` を立てる。これで:
- 同フレーム内で **WallKickAction より後に Tick される JumpAction** は `jumpRequested == false` を見て早期 return
- jumpsUsed が無駄消費されない

ただし **inventory.owned の順序依存** がある。WallKick が Jump より後ろに並ぶと Jump が先に発火して空中ジャンプ回数が消費される。Shop 購入順 (Jump → WallKick) でこの順になりやすいので、注意点として残る。

### flipX 制御

原画 `wall-jump.png` は「右の壁を蹴って左に飛ぶ」向きで描かれている (キャラ本体は左向き)。既存の bridge ロジック (`flipX = vx < 0`) は「原画が右向き」を前提とした逆ルールのため、wall-jump 中のみオーバーライドが必要:

| 状況 | `wallKickSide` | 期待 | flipX |
|---|---|---|---|
| 右壁を蹴る (左に飛ぶ) | +1 | 原画のまま (左向き) | false |
| 左壁を蹴る (右に飛ぶ) | -1 | 反転 (右向き) | true |

→ `flipX = (wallKickSide < 0)` という単純な式に落ちる。

vx ベースだと Windup フェーズ (vx ≒ 0.1 × wallSide で微小) で誤判定するので、`wallKickSide` を直接コンテキスト経由で渡す方式を採用。

### スプライトを 11 フレーム動かす vs 1 枚固定

Sliding は 1 枚固定だったが、WallKick は **明確な状態遷移 (壁張り付き → 蹴り出し → 飛行 → 着地)** があるためフレームアニメが効く。アニメ時間 (0.833s) と物理状態時間 (windup 0.15s + postKick 0.6s = 0.75s) はほぼ一致させてあるので、絵と挙動がズレない。

## MCP for Unity 駆動の作業フロー

本実装は手作業を最小化するため、すべて MCP 経由で行った:

1. **コード生成** — Edit/Write ツールで `.cs` / `.asset` を直接編集
2. **コンパイル確認** — `mcp__UnityMCP__refresh_unity` + `read_console`
3. **スプライト解析** — `execute_code` (codedom) で `Texture2D.GetPixels32()` を使った連結成分検出
4. **再スライス** — `TextureImporter.spritesheet` API で 11 フレームの SpriteMetaData を一括設定
5. **アニメクリップ作成** — `AnimationUtility.SetObjectReferenceCurve` で SpriteRenderer.m_Sprite カーブを設定
6. **Animator 編集** — `AnimatorController.AddParameter` / `AddState` / `AddAnyStateTransition` で state machine を構築
7. **動作検証** — `EditorApplication.isPlaying` で play mode に入り、`animator.Play("WallKick")` で WallKick state に遷移して各時刻の sprite を取得して確認

特に再スライスは 124 個のオートスライス結果から 11 個の主要フィギュアを自動抽出するロジック (面積降順で上位 12 を取って 1 番大きいタイトル除外、Y で行分割、X で列ソート) を書いた。手作業で 1 つずつ rect 入力するより遥かに早い。
