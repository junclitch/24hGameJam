# しゃがみアクション 実装まとめ

実装日: 2026-05-05 (09:00 〜 12:30 頃)
対象シーン: `Assets/Scenes/ScrollAction.unity`
関連: [是正報告書](../correction/20260505-114000-crouch-feet-floating.md)

## 概要

ScrollAction の PlayerAction インベントリ系に「しゃがみ」アクションを追加した。`base-action.png` の 4 段目「しゃがみ」フレームを使い、以下を実現:

- 下方向入力 (`↓` / `S`) かつ接地中にしゃがみ姿勢へ遷移
- 当たり判定 (BoxCollider2D) を縦方向に縮小、底面 (足元) は固定
- アニメーションは frame 0→4 のワンショット (姿勢を低くする動作) → 最低姿勢で保持

実装中、スプライトシートの自動スライスが印刷参考用ラベルを sprite rect に含めていた問題を発見・是正。jump アニメにも同じ問題があったため横展開で修正した。

## 成果物

### 1. PlayerAction (新規)
`Assets/ScrollAction/Scripts/Actions/CrouchAction.cs`

- `[CreateAssetMenu]` 付き ScriptableObject (既存の HorizontalMove / Jump / Evasion 等と同じ A+B+C パターン)
- `OnFixedTick` で:
  - `ctx.crouchPressed && ctx.isGrounded` のときだけ発動
  - `ctx.rb.linearVelocity.x = 0` で水平移動を抑制 (Y 速度は維持)
  - BoxCollider2D の `size.y` を `crouchedSizeY` (.asset で設定) に縮小、`offset.y` を再計算して**底面を固定**
  - `ctx.isCrouching = true` をセットし、PlayerController 経由で AnimatorBridge へ橋渡し
- 入力解放時に元のサイズへ復元 (初回 Tick で元値をキャッシュ → SO 使い回しに対応)
- `OnRespawn` / `OnSessionInit` で内部状態リセット

```csharp
private void ApplyShrunkCollider(BoxCollider2D box)
{
    float bottomLocal = originalOffsetY - originalSizeY * 0.5f;
    box.size = new Vector2(box.size.x, crouchedSizeY);
    box.offset = new Vector2(box.offset.x, bottomLocal + crouchedSizeY * 0.5f);
    currentlyShrunk = true;
}
```

### 5. Animator Controller 更新
`Assets/ScrollAction/Animations/Player.controller`

- パラメータ追加: `IsCrouching` (bool)
- ステート追加: `Crouch` (motion: crouch.anim)
- 遷移追加:
  - Idle → Crouch (`IsCrouching == true`)
  - Crouch → Idle (`IsCrouching == false`)
- 既存の Walk → Crouch 遷移は不要 (CrouchAction が水平速度を 0 にするため Walk は自動的に Idle へ抜ける → そこから Crouch へ)

### 6. 入力・状態橋渡しの拡張

**`PlayerActionContext.cs`**
- `bool crouchPressed` 追加 (Update 側で集約)
- `bool isCrouching` 追加 (CrouchAction が立てる出力)

**`PlayerController.cs`**
- Update で `kb.downArrowKey.isPressed || kb.sKey.isPressed` を毎フレーム読み (継続入力)
- FixedUpdate で ctx に転送、Tick 後に `IsCrouching` プロパティへ反映

**`PlayerAnimatorBridge.cs`**
- `IsCrouchingHash` キャッシュ追加
- 既存の `IsGrounded` / `Speed` / `VerticalSpeed` と同様、毎 Update で `controller.IsCrouching` を Animator に流す

## スプライト rect 補正 (重要)

`base-action.png` の crouch / jump 行は印刷参考用に各フレームの下に「01」「02」「03」… のインデックスラベルが描き込まれており、Unity 自動スライスが**ラベルを sprite rect に含めて切り出していた**。

pivot=(0.5, 0) は rect の底辺中央なので、これにより:
- ラベル位置 (= 絵の足元のずっと下) が pivot として扱われた
- `sr.bounds.bottom` を地面に合わせても、絵自体は地面から数十 px 浮いた位置に描画されていた

### 補正対象 (15 フレーム)

| sprite | 元 height | 修正後 height | 削った行数 |
|---|---|---|---|
| base_crouch_00 | 190 | 154 | 36 |
| base_crouch_01 | 167 | 131 | 36 |
| base_crouch_02 | 139 | 109 | 30 |
| base_crouch_03 | 137 | 103 | 34 |
| base_crouch_04 | 133 | 105 | 28 |
| base_crouch_05 | 179 | 148 | 31 |
| base_crouch_06 | 169 | 145 | 24 |
| base_crouch_07 | 181 | 151 | 30 |
| base_jump_00 | 127 | 104 | 23 |
| base_jump_01 | 163 | 128 | 35 |
| base_jump_02 | 195 | 123 | 72 |
| base_jump_03 | 221 | 113 | **108** |
| base_jump_04 | 213 | 118 | 95 |
| base_jump_05 | 166 | 113 | 53 |
| base_jump_06 | 132 | 109 | 23 |

特に `base_jump_03` は 108 行 (1.08 world unit) もキャラがピボットから上方にずれていた。idle / walk 行はラベルが付いていなかったため影響なし。

補正は `ISpriteEditorDataProvider` 経由で `base-action.png.meta` の rect を直接更新。各フレームの「ラベルとキャラ本体の間の最長空白行」を画素解析で検出し、それより下を rect から除外。

## 動作確認方法 (本セッション中に確立した検証プロトコル)

### 静的レイアウト
- `Camera.Render()` 同期描画 → `ReadPixels` → `EncodeToPNG` で十分

### 動的・アニメーション
- 単発スクショで判定しない。`anim.Update(dt) → Camera.Render() → 保存` をフレームごとループ、または ffmpeg で動画化、もしくはユーザー実機録画
- `ScreenCapture.CaptureScreenshot` (非同期) は使わない (撮影タイミングが意図とずれる)
- `transform.position` を強制配置して即撮影しない (物理が落ち着かない / 接地判定未走行)

### スプライト関連の追加確認
- 新規スプライトをスライスしたら **各フレームを個別 extract して目視** (ラベル・余白の混入確認)
- `sr.bounds.bottom` は **rect 底辺**であり**絵の足元**とは限らないことを常に意識
- 不安なら debug marker (赤クアッド) を `sr.bounds.bottom` に配置して可視化

詳細は `document/unity-coding-conventions.md` の「確認・撮影プロトコル」と memory `feedback_verify_colliders.md` を参照。

## 反省・教訓

本セッションでは「足が浮く」現象に対して、Claude が以下の失敗をした:

1. 数値検証 (`sr.bounds.bottom == ground top`) だけで完了判定し、**絵の足元**との一致を確認しなかった
2. 強制配置 (Player.y=1.9 = 空中位置) で撮ったスクショを根拠に「OK」と判断
3. ユーザーが「足浮いてますよね」と複数回指摘した時に、自分の検証手法を疑わず「数値上は浮いていない」と反論
4. ユーザー提供の動画でしゃがみが画面に表示されていなかったのに、初期は「環境問題」と返答
5. crouch のラベル混入を見つけた後、同じ素材の jump にも同じ問題がある可能性を即座に検討しなかった (ユーザーから「他のアニメーションも確認して」と促されてようやく)

これらの教訓は是正報告書 (`document/correction/20260505-114000-crouch-feet-floating.md`) と memory に記録済み。
