## 概要

ScrollAction の PlayerAction インベントリ系に「転がる」アクションを追加した。`add2-action.png` の 2 行目 (でんぐり返し) のフレームを使い、以下を実現:

- **Q キー長押し**で発動 (押している間継続)
- 発動中は **BoxCollider2D をしゃがみと同じ高さに縮める** (底面=足元固定)
- 押している間 **入力方向に rollingSpeed (歩きより少し速い) で水平移動**
- 3 段階アニメ:
  - **RollingStart**: フレーム 1-4 (`_81`〜`_84`) を 1 回だけ再生
  - **RollingLoop**: フレーム 5 (`_85`) を 1 枚固定し、Z 軸回転 + 位置補正で「目を不動点にして回す」
  - **RollingEnd**: フレーム 7-10 (`_87`〜`_90`) を 1 回再生 → Idle へ
- ショップで売買可能 (Shop catalog に追加)
- **左右両対応**: 進行方向 (flipX) に応じて回転方向と x 補正の符号を反転

## 行番号確認の経緯

ユーザー初回依頼は「4 行目」だったが、`add2-action.png` の 4 行目は **壁キック** で「転がる」絵柄ではなかった。

実際の行構成:
- 1 行目: 爆弾ジャンプ
- 2 行目: でんぐり返し ← **転がる絵柄**
- 3 行目: 大ジャンプ
- 4 行目: 壁キック

ユーザーに「絵柄が指定と合わない」旨を確認したところ「でんぐり返し」(2 行目) で確定。**「絵と要件の不整合は推測せず確認する」** を実行。

## アーキテクチャ概観

### Player.prefab の階層構造 (本実装で変更)

```
Player    (localScale = 0.8, 1.2)   ← 物理ルートの非一様スケール (BoxCollider2D 等)
  └─ Visual    (localScale = 1.25, 0.83)   ← Player.scale を打ち消す counter-scale。Animator 設置
     └─ Spinner   (localScale = 1, 1, 1)    ← 回転 + 位置補正の専任 (本実装で新設)
        └─ (SpriteRenderer)
```

**ポイント**: `Spinner` の上で `Visual.scale × Player.scale = (1, 1)` が打ち消しあい、Spinner より上の合成スケールが**単位行列**になる。これにより Spinner で回転させても shear (せん断歪み) が発生しない。Visual で回転していた当初は、`Player.scale (0.8, 1.2)` と `Visual.scale (1.25, 0.83)` の組み合わせで shear が発生し、絵の比率が回転中に変化していた。

### A+B+C パターン (CLAUDE.md 規約)

- **A. イベント駆動**: `ctx.rollingHeld` (入力) と `ctx.isRolling` (実行中フラグ) を `PlayerActionContext` 経由で疎結合に橋渡し
- **B. ScriptableObject**: `Rolling.asset` に `rollingSpeed` / `rollingSizeY` / `buyPrice` / `sellPrice` を分離
- **C. 状態管理**: Animator FSM (`RollingStart` / `RollingLoop` / `RollingEnd` の 3 ステート連鎖)

## 成果物

### 1. PlayerAction (新規)

`Assets/ScrollAction/Scripts/Actions/RollingAction.cs`

- `[CreateAssetMenu]` 付き ScriptableObject (Crouch / Sliding と同じ A+B+C パターン)
- `OnFixedTick`:
  - `wantRoll = ctx.rollingHeld && ctx.isGrounded` で発動条件チェック
  - 発動中: `ApplyShrunkCollider(box)` でコライダー縮小、`ctx.rb.linearVelocity = (ctx.inputX × rollingSpeed, vy)` で水平速度を上書き、`ctx.isRolling = true` をセット
  - 発動外で `currentlyShrunk` なら `RestoreCollider(box)` で復元
- `OnRespawn`: 物理復元はせずフラグだけ落とす (CrouchAction と同方針)

```csharp
bool wantRoll = ctx.rollingHeld && ctx.isGrounded;
if (wantRoll)
{
    ApplyShrunkCollider(box);
    ctx.rb.linearVelocity = new Vector2(ctx.inputX * rollingSpeed, ctx.rb.linearVelocity.y);
    ctx.isRolling = true;
}
else if (currentlyShrunk)
{
    RestoreCollider(box);
}
```

## 回転 + 位置補正の数学

### 不動点 (目の位置)

`add2-action_85` (94×67px, ppu=100, pivot at (47, 0) = 足元中央) の目の visual center:

- sprite-pixel: `(67.67, 43.92)`  (連結成分 bbox 推定 + 実機レンダリング empirical 補正)
- sprite-local 世界単位: `(0.20, 0.435)` (pivot からのオフセット)

`Inspector.rollingEyeOffsetSprite = (0.20, 0.435)` として設定。

### 補正式 (右向き、`flipX=false`)

回転角 ψ ∈ [0°, 360°)、Spinner.localRotation.z = -ψ (時計回り):

```
Spinner.localPosition.x = ex × (1 - cos ψ) - ey × sin ψ
Spinner.localPosition.y = ey × (1 - cos ψ) + ex × sin ψ
```

ここで `ex = 0.20`, `ey = 0.435`。

導出: 目を世界座標で `(player + 0.8 × ex, player + 1.2 × (-0.5 + ey))` に固定したい。Spinner より上のスケールが単位行列なので Spinner-local では `(ex, ey)` 不動を満たす条件 `Spinner.localPos + R(ψ) × (ex, ey) = (ex, ey)` を解いた。

### 左向き対応 (`flipX=true`)

flipX で sprite が pivot 軸対称にミラーされ、目の visual 位置が `(-0.20, +0.435)` (Spinner-local) に移る。回転を逆向き (反時計回り) にし、x 補正の符号を反転すれば不動点維持:

```
sign = flipX ? -1 : +1
locX = sign × (ex × (1 - cos ψ) - ey × sin ψ)
locY = ey × (1 - cos ψ) + ex × sin ψ      // 不変
rotZ = -sign × ψ
```

vy が左右で同一なのは、`ex × sin` 項の符号反転 (flipX) と回転方向反転 (sign) が打ち消しあうため。数学的に確認済み。

## 動作の予測値

| シナリオ | 期待される挙動 |
|---|---|
| Q を押す (接地中) | RollingTrigger 発火 → RollingStart 4 コマ (0.333s) → RollingLoop に入って回転開始 |
| Q を押し続ける | RollingLoop で 0.5s/周の回転、目が世界座標で不動、絵の比率も保たれる、`inputX × 9 units/s` で水平移動 |
| Q を離す | IsRolling=false → RollingLoop → RollingEnd 遷移開始の瞬間、Spinner が (0,0,0) にスナップ → RollingEnd 4 コマ (0.333s) → Idle |
| 押し直し | 立ち上がりエッジで再度 SetTrigger → Any State → RollingStart |
| 空中で Q | wantRoll=false (isGrounded で gating)、何も起きない |
| ショップ売買 | catalog の最後尾。35 で買って 35 で売れる |
